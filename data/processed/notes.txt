### 1. Introduction to Memory Systems

The memory system is a fundamental component of any computer, essential for storing and retrieving various types of information required by the processor. Its performance significantly impacts the overall speed and efficiency of the computer.

#### 1.1 Memory System Overview
*   **Role:** Responsible for storing program instructions, input data, intermediate results, and output data.
*   **Performance Impact:** Computer system performance is a function of both processor speed and memory system performance, including their interface.

#### 1.2 Processor-Memory Gap
*   **Problem:** Modern processors operate at significantly higher speeds (GHz) compared to main memory (DRAM, ns access times).
*   **Consequence:** This growing speed disparity causes the processor to frequently wait for data from memory, leading to idle CPU cycles and reduced system performance.
*   **DRAM vs. CPU:** DRAM is much slower than CPU clock speed.

#### 1.3 Memory System Challenge
*   **Ideal Memory:** An ideal memory would be **fast**, **large in capacity**, and **inexpensive**.
*   **Practical Impossibility:** It is technologically and economically impossible to achieve all three characteristics simultaneously in a single memory technology.
*   **Solution:** Designers employ a **memory hierarchy** to balance these conflicting tradeoffs.

#### 1.4 Memory Hierarchy
*   **Concept:** Organizes memory into multiple levels based on speed, cost, and size. Faster, smaller, and more expensive memory levels are placed closer to the CPU, while slower, larger, and cheaper levels are further away.
*   **Principle:** Frequently accessed data is stored in faster, higher levels of the hierarchy.
*   **Typical Levels (from fastest/smallest to slowest/largest):**
    1.  **Registers:** Smallest, fastest memory, directly within the CPU, used for immediate data access during computation.
    2.  **Cache Memory (SRAM):** Small, fast memory levels (L1, L2, L3) located on or very close to the CPU chip. Stores frequently used instructions and data.
    3.  **Main Memory (DRAM):** Larger, slower than cache, holds active programs and data.
    4.  **Secondary Storage (SSDs, Hard Drives):** Largest, slowest, non-volatile storage for long-term data retention and virtual memory.

*   **Characteristics of Memory Hierarchy Levels:**

| Characteristic | SRAM (Cache) | DRAM (Main Memory) | SSD (Secondary) | HDD (Secondary) |
| :------------- | :----------- | :----------------- | :-------------- | :-------------- |
| **Technology**   | SRAM         | DRAM               | Flash Memory    | Magnetic Disk   |
| **Capacity**   | Small (KB-MB) | Medium (GB)        | Large (GB-TB)   | Very Large (TB) |
| **Speed**      | Very Fast    | Fast               | Moderate        | Slow            |
| **Price / GB** | High ($100)  | Medium ($3)        | Low ($0.10)     | Very Low ($0.03)|
| **Access Time (ns)** | 0.2 - 3      | 10 - 50            | 20,000          | 5,000,000       |
| **Bandwidth (GB/s)** | 100+         | 30                 | 0.05 - 3        | 0.001 - 0.1     |

    **ЁЯОп Important Exam Point:** Be able to describe the memory hierarchy and explain the tradeoffs at each level.

#### 1.5 Locality of Reference
*   **Definition:** Programs tend to access memory in predictable, non-random patterns. Exploiting this property is crucial for the efficiency of memory hierarchies.
*   **Types of Locality:**
    *   **a. Temporal Locality:**
        *   **Concept:** If a memory location is accessed, it is likely to be accessed again in the near future.
        *   **Examples:** Loop variables, frequently called functions, instruction fetches within a loop.
        *   **Exploitation:** Keep recently accessed data in higher (faster) levels of the memory hierarchy (e.g., cache).
    *   **b. Spatial Locality:**
        *   **Concept:** If a memory location is accessed, memory locations *near* it are likely to be accessed soon.
        *   **Examples:** Array traversals, sequential instruction execution, structure fields.
        *   **Exploitation:** When data is accessed, bring nearby data (a block or line) into higher levels of the memory hierarchy along with the requested data.

    **тнР Frequently Asked University Exam Topic:** Explain the concept of locality of reference with examples for both temporal and spatial locality.

#### 1.6 Memory Interface
*   **Communication:** The processor communicates with memory using several buses and control signals.
*   **Components:**
    *   **Address Bus:** Specifies the unique memory location to be accessed.
    *   **Data Bus:** Carries the actual data being transferred between the processor and memory.
    *   **Control Signals:** Indicate the type of operation (e.g., Read Enable (RE), Write Enable (WE), Clock (CLK)).

#### 1.7 Cache Memory (Introduction)
*   **Purpose:** A small, fast memory placed between the CPU and main memory to bridge the processor-memory speed gap.
*   **Technology:** Built using Static Random Access Memory (SRAM), which is faster and more expensive than DRAM.
*   **Benefit:** Stores frequently used instructions and data, significantly reducing average memory access time and improving overall system performance.

### 2. Memory Performance Analysis

Understanding memory system performance is critical for designing efficient computer systems. Key metrics include hit rate, miss rate, and Average Memory Access Time (AMAT).

#### 2.1 Hit and Miss
When the processor requests data, the memory hierarchy is checked from the fastest level downwards.

*   **Hit:**
    *   **Definition:** Occurs when the requested data (instruction or operand) is found in the *current level* of the cache (or memory hierarchy).
    *   **Outcome:** Fast data retrieval from the higher-speed memory.
*   **Miss:**
    *   **Definition:** Occurs when the requested data is *not found* in the current level of the cache.
    *   **Outcome:** The data must be fetched from a lower (slower) level of the memory hierarchy, incurring a **miss penalty**.

#### 2.2 Hit Rate and Miss Rate
These metrics quantify the effectiveness of a cache level.

*   **a. Hit Rate:**
    *   **Definition:** The fraction of memory accesses that are found in the cache.
    *   **Formula:**
        `Hit Rate = Number of Hits / Total Memory Accesses`
        `Hit Rate = 1 тАУ Miss Rate`
    *   **Significance:** A higher hit rate indicates better cache performance and, consequently, better system performance.

*   **b. Miss Rate:**
    *   **Definition:** The fraction of memory accesses that are *not* found in the cache.
    *   **Formula:**
        `Miss Rate = Number of Misses / Total Memory Accesses`
        `Miss Rate = 1 тИТ Hit Rate`
    *   **Significance:** A lower miss rate improves system performance by reducing the number of costly accesses to slower memory levels.

#### 2.3 Average Memory Access Time (AMAT)
*   **Definition:** AMAT represents the average time taken for the processor to access a piece of data from the memory system. It accounts for both cache hits and misses, and the penalties associated with misses.
*   **Importance:** AMAT is a crucial metric for evaluating memory system efficiency. A lower AMAT indicates faster average access time and better overall system performance.

*   **Formulas:**
    *   **For a Single-Level Cache:**
        `AMAT = Cache Access Time + (Miss Rate ├Ч Main Memory Access Time)`
        Where:
        *   `Cache Access Time`: Time to access data if it's in the cache (hit time).
        *   `Miss Rate`: Probability of a cache miss.
        *   `Main Memory Access Time`: Time to retrieve data from main memory *after* a cache miss (miss penalty).

    *   **For a Multi-Level Cache (e.g., L1 and L2):**
        `AMAT = t_L1 + MR_L1 ├Ч (t_L2 + MR_L2 ├Ч t_Memory)`
        Where:
        *   `t_L1`: Level 1 cache access time.
        *   `MR_L1`: L1 cache miss rate.
        *   `t_L2`: Level 2 cache access time.
        *   `MR_L2`: L2 cache miss rate (this is the local miss rate of L2, i.e., misses from L1 that then miss in L2).
        *   `t_Memory`: Main memory access time (miss penalty from L2).

    **тнР Frequently Asked University Exam Topic:** Derive and explain the AMAT formula for both single and multi-level caches. Provide an example.

*   **Concept:** Even if `Cache Access Time` (tL1) is very small, a significant `Miss Rate` (MR_L1) combined with a large `Miss Penalty` (`t_L2 + MR_L2 ├Ч t_Memory`) can drastically increase the `AMAT`. Therefore, reducing the miss rate is crucial for performance.

#### 2.4 AmdahlтАЩs Law
*   **Statement:** Amdahl's Law states that the performance improvement of a system is limited by the portion of the system that is *not* improved.
*   **Meaning for Memory Systems:** Improving memory performance will significantly increase overall system speed *only if* memory access forms a large portion of the total execution time. If the processor spends most of its time on computation rather than waiting for memory, memory performance improvements will have diminishing returns on overall system speed.
*   **Formula (General):**
    `Speedup = 1 / [(1 - FractionImproved) + (FractionImproved / SpeedupImproved)]`
    Where:
    *   `FractionImproved`: Proportion of execution time that benefits from the improvement.
    *   `SpeedupImproved`: How much faster the improved part is.

    **ЁЯОп Important Exam Point:** Understand how Amdahl's Law applies to memory system optimization.

#### 2.5 Examples of Memory Performance Analysis

*   **Example 1: Cache Hit and Miss Rates**
    *   **Problem:** A program has 2,000 loads and stores. 1,250 of these data values are found in cache. The rest are supplied by other levels of the memory hierarchy. What are the cache hit and miss rates?
    *   **Solution:**
        *   Total Memory Accesses = 2,000
        *   Number of Hits = 1,250
        *   Number of Misses = Total Memory Accesses - Number of Hits = 2,000 - 1,250 = 750
        *   `Hit Rate = Number of Hits / Total Memory Accesses = 1,250 / 2,000 = 0.625` (or 62.5%)
        *   `Miss Rate = Number of Misses / Total Memory Accesses = 750 / 2,000 = 0.375` (or 37.5%)
        *   `Check: Miss Rate = 1 - Hit Rate = 1 - 0.625 = 0.375`

*   **Example 2: Average Memory Access Time (AMAT)**
    *   **Problem:** Suppose a processor has two levels of hierarchy: cache and main memory. The cache access time (`t_cache`) is 1 cycle, and main memory access time (`t_MM`) is 100 cycles. What is the AMAT for the program from Example 1?
    *   **Given from Example 1:** Miss Rate (`MR_cache`) = 0.375
    *   **Solution:**
        *   Using the single-level cache AMAT formula:
            `AMAT = t_cache + MR_cache ├Ч t_MM`
            `AMAT = 1 cycle + (0.375 ├Ч 100 cycles)`
            `AMAT = 1 cycle + 37.5 cycles`
            `AMAT = 38.5 cycles`

    *   **Observation:** Even with a fast cache hit time (1 cycle), a relatively high miss rate (37.5%) combined with a large miss penalty (100 cycles) results in a significantly increased average access time (38.5 cycles). This highlights the importance of reducing the miss rate.

### 3. Caches тАУ Basic Concepts

Cache memory is a critical component in modern computer architecture, specifically designed to bridge the performance gap between fast CPUs and slower main memory.

#### 3.1 Need for Cache Memory
*   **Processor Speed:** Modern processors operate at extremely high clock frequencies.
*   **Main Memory Speed:** Main memory (DRAM) is significantly slower than the CPU.
*   **Performance Gap:** This speed mismatch causes the CPU to frequently stall, waiting for data from main memory, leading to:
    *   CPU remaining idle.
    *   Increased program execution time.
    *   Overall poor system performance.
*   **Solution:** Cache memory mitigates this by storing frequently accessed data closer to the processor, reducing the average memory access delay.

#### 3.2 Cache in Memory Hierarchy
*   **Position:** Cache memory acts as an intermediate, faster layer between the CPU and main memory.
*   **Working Principle:**
    1.  The processor requests data (instruction or operand).
    2.  The cache is checked first.
    3.  **If data is present (Cache Hit):** Data is quickly supplied to the CPU from the cache.
    4.  **If data is not present (Cache Miss):** The data is fetched from the next lower level (e.g., main memory), brought into the cache, and then supplied to the CPU.

#### 3.3 Cache Hit and Cache Miss (Detailed)

*   **A. Cache Hit:**
    *   **Definition:** The event where the CPU finds the requested data in the cache.
    *   **Advantages:**
        *   **Faster Data Access:** Data is retrieved from the high-speed cache.
        *   **Reduced Memory Latency:** The delay in accessing memory is significantly lower.
        *   **Improved CPU Performance:** CPU spends less time waiting, leading to higher throughput.

*   **B. Cache Miss:**
    *   **Definition:** The event where the CPU does not find the requested data in the cache.
    *   **Effects:**
        *   **Increased Access Time:** Data must be fetched from main memory (or lower levels), which is much slower.
        *   **Miss Penalty:** The time taken to fetch the data from a slower level and transfer it to the cache.
        *   **Reduced System Performance:** CPU stalls during the fetch, lowering overall system efficiency.

#### 3.4 Cache Terminology
To understand cache organization, specific terms are essential:

*   **I. Cache Capacity (C):**
    *   **Definition:** The total amount of data that the cache can store.
    *   **Measurement:** Typically measured in Kilobytes (KB) or Megabytes (MB).
    *   **Impact:** Larger capacity generally leads to lower miss rates (can hold more data), but increases cost and access time.

*   **II. Block Size (b) / Cache Line Size:**
    *   **Definition:** The fixed amount of data transferred as a single unit between memory and cache at one time. A block is the smallest unit of transfer.
    *   **Measurement:** Usually in words or bytes (e.g., 64 bytes).
    *   **Impact:**
        *   Larger block size exploits spatial locality better (brings more nearby data).
        *   Too large a block size can lead to fetching unneeded data (increasing miss penalty) and replacing potentially useful data.

*   **III. Number of Blocks (B):**
    *   **Definition:** The total number of blocks (or cache lines) that the cache can hold.
    *   **Formula:** `B = C / b`
        *   *Example:* If Cache Capacity (C) = 8 words and Block Size (b) = 1 word, then `B = 8 / 1 = 8 blocks`.

*   **IV. Associativity (N):**
    *   **Definition:** The number of blocks (cache lines) within each set of a set-associative cache. It determines the flexibility of data placement within the cache.
    *   **Impact:** Higher associativity generally reduces conflict misses but increases hardware complexity and access time.

*   **V. Number of Sets (S):**
    *   **Definition:** The total number of unique sets in the cache.
    *   **Mapping:** Each memory address maps to exactly one cache set.
    *   **Formula:** `S = B / N` (where B is total blocks, N is associativity)
        *   *Example (using B=8 from above):*
            *   Direct Mapped Cache (N=1): `S = 8 / 1 = 8 sets`
            *   2-Way Set Associative Cache (N=2): `S = 8 / 2 = 4 sets`
            *   Fully Associative Cache (N=B): `S = 8 / 8 = 1 set`

    **ЁЯОп Important Exam Point:** Be able to calculate B and S given C, b, and N.

#### 3.5 How is Data Found in Cache? (Cache Organization Types)
Cache memory is organized to efficiently map main memory addresses to cache locations. Caches are primarily classified into three types based on their mapping scheme:

*   **A. Direct Mapped Cache:**
    *   **Concept:** Each memory block maps to *exactly one* specific cache location (set).
    *   **Organization:** Each set contains one block (`N=1`). The number of sets `S` is equal to the total number of blocks `B`.
    *   **Advantages:** Simple hardware, fast access time due to no complex search.
    *   **Disadvantages:** Suffers from high **conflict misses** (when frequently accessed data items map to the same cache location, repeatedly evicting each other).

*   **B. N-Way Set Associative Cache:**
    *   **Concept:** A memory block can be placed in *any* of the `N` blocks within its mapped set.
    *   **Organization:** Each set contains `N` blocks (`N > 1`). The total number of sets `S = B / N`.
    *   **Advantages:** Better performance than direct-mapped cache by reducing conflict misses, moderate hardware complexity.
    *   **Disadvantages:** More complex hardware than direct-mapped (requires comparing `N` tags simultaneously), slightly slower access.

*   **C. Fully Associative Cache:**
    *   **Concept:** A memory block can be placed in *any* available block anywhere in the entire cache. All cache blocks belong to a single set (`S=1`, `N=B`).
    *   **Organization:** The entire cache is effectively one large set.
    *   **Advantages:** Offers maximum flexibility in data placement, resulting in the lowest possible conflict misses.
    *   **Disadvantages:** Extremely complex hardware (requires comparing every cache block's tag simultaneously), very expensive, and slow for large caches. Primarily used for small caches like TLBs.

    **Comparison Table: Cache Mapping Techniques**

| Feature           | Direct Mapped Cache           | N-Way Set Associative Cache      | Fully Associative Cache           |
| :---------------- | :---------------------------- | :------------------------------- | :-------------------------------- |
| **Associativity (N)** | 1 (1 block per set)           | N (N blocks per set, N > 1)      | B (all blocks in one set)         |
| **Number of Sets (S)** | B (Total Blocks)              | B / N                            | 1                                 |
| **Mapping Rule**  | Memory Block -> Specific Cache Block | Memory Block -> Specific Set, Any Block within Set | Memory Block -> Any Cache Block |
| **Flexibility**   | Low                             | Moderate                         | High                              |
| **Hardware Complexity** | Simple                          | Moderate                         | High (for large caches)           |
| **Cost**          | Low                             | Moderate                         | High                              |
| **Hit Rate**      | Lowest (high conflict misses) | Better than Direct Mapped        | Highest (lowest conflict misses)  |
| **Access Time**   | Fastest                         | Moderate                         | Slowest (for large caches)        |

### 4. Direct Mapped Cache (Detailed)

The direct mapped cache is the simplest cache organization. Understanding its structure and operation is fundamental to comprehending cache memory.

#### 4.1 Organization
*   **Mapping:** Each block of main memory maps to exactly *one* specific cache set.
*   **Sets:** A direct mapped cache has one block in each set (`N=1`), so the number of sets `S` is equal to the total number of cache blocks `B` (`S = B`).
*   **Block Division:** Main memory is conceptually divided into blocks of the same size as cache blocks (`b`).
*   **Index Mapping:** Main memory block `j` maps to cache set `(j MOD S)`. For example, if `S=8`, memory block 0 maps to cache set 0, block 1 to set 1, ..., block 7 to set 7, and block 8 maps back to set 0. This "wraps around" the cache.
*   **Single Location:** Each memory address has only one possible location in the cache.

#### 4.2 Address Breakdown in Direct Mapped Cache
To access data in a direct mapped cache, a memory address is partitioned into three fields:

`Memory Address = [ Tag | Set Index | Byte Offset ]`

1.  **Byte Offset:**
    *   **Bits:** The `logтВВ(b)` least significant bits of the address, where `b` is the block size in bytes.
    *   **Purpose:** Specifies the exact byte within a cache block (or word within a block, if accessing words).
    *   **Note:** For word-aligned addresses and word access, these bits are often 00 (for 4-byte words).

2.  **Set Index:**
    *   **Bits:** The next `logтВВ(S)` bits of the address, where `S` is the total number of sets in the cache.
    *   **Purpose:** Used to select one specific cache set from `S` available sets. This directly determines which cache line *could* hold the requested data.

3.  **Tag:**
    *   **Bits:** The remaining most significant bits of the address.
    *   **Purpose:** Used to uniquely identify which specific memory block is currently stored in the selected cache set. Since multiple memory blocks can map to the same set (due to the `MOD S` operation), the tag differentiates them.

#### 4.3 Cache Set Contents
Each entry (or block) in a cache set stores the following:
*   **Data:** The actual data (a block of `b` bytes) fetched from main memory.
*   **Tag:** The tag portion of the memory address that currently resides in this cache block.
*   **Valid Bit:** A single bit indicating whether the cache entry contains meaningful, valid data (1 = valid, 0 = invalid/empty). This is essential for distinguishing empty cache lines or lines containing stale data.

#### 4.4 Direct Mapped Cache Hardware
The direct mapped cache is typically implemented using an SRAM array and comparison logic.

*   **Structure:** The cache is an `S`-entry SRAM array, where each entry corresponds to one cache set. Each entry stores `Data`, `Tag`, and `Valid Bit`.
*   **Access Steps:**
    1.  **Index Extraction:** The `Set Index` bits from the requested memory address are used to select a specific entry (cache line) in the SRAM array.
    2.  **Read Cache Line:** The `Data`, `Tag`, and `Valid Bit` stored at that selected cache entry are read simultaneously.
    3.  **Tag Comparison:** The `Tag` bits from the requested memory address are compared with the `Tag` stored in the cache entry.
    4.  **Valid Bit Check:** The `Valid Bit` is checked to ensure the data is current.
    5.  **Hit/Miss Determination:**
        *   **Cache Hit:** If the stored `Tag` matches the requested address `Tag` AND the `Valid Bit` is 1, a cache hit occurs. The `Data` from the cache is returned to the processor, using the `Byte Offset` to select the specific word/byte within the block.
        *   **Cache Miss:** If the `Tag` does not match OR the `Valid Bit` is 0, a cache miss occurs. The processor must then fetch the entire block of data from main memory, store it into the appropriate cache set (overwriting the old content), update its `Tag` and set its `Valid Bit` to 1, and finally return the requested data to the processor.

    **Advantages:** Only one tag comparison is needed per access, making it fast and simple to implement.
    **Disadvantages:** Prone to conflict misses, as discussed earlier.

#### 4.5 Examples of Direct Mapped Cache Operation

*   **Example 4: Temporal Locality with a Direct Mapped Cache**
    *   **Scenario:** An 8-word direct mapped cache (block size = 1 word). Cache is initially empty (all valid bits = 0).
        *   Cache Capacity (C) = 8 words
        *   Block Size (b) = 1 word
        *   Number of Blocks (B) = C/b = 8/1 = 8 blocks
        *   Number of Sets (S) = B/N = 8/1 = 8 sets (since Direct Mapped, N=1)
        *   Assume 32-bit addresses and word-aligned access, so Byte Offset = 2 bits (for 4-byte words). Set Index = logтВВ(S) = logтВВ(8) = 3 bits. Tag = 32 - 2 - 3 = 27 bits.
    *   **RISC-V Loop:**
        ```assembly
        addi s0, zero, 5  # s0 = 5 (loop counter)
        addi s1, zero, 0  # s1 = 0 (base address for loads)
        LOOP: beq s0, zero, DONE
         lw s2, 4(s1)     # Load word from address 0x4
         lw s3, 12(s1)    # Load word from address 0xC
         lw s4, 8(s1)     # Load word from address 0x8
         addi s0, s0, тИТ1  # Decrement loop counter
         j LOOP
        DONE:
        ```
    *   **Analysis:**
        *   **Total Accesses:** The loop runs 5 times. Each iteration has 3 load instructions. So, 5 * 3 = 15 total memory accesses.
        *   **Address Mapping:**
            *   Address 0x4: `[Tag (0x0...0) | Index (001) | Offset (00)]` -> maps to Set 1.
            *   Address 0xC: `[Tag (0x0...0) | Index (011) | Offset (00)]` -> maps to Set 3.
            *   Address 0x8: `[Tag (0x0...0) | Index (010) | Offset (00)]` -> maps to Set 2.
            *   *Note:* The tags are all 0 because the upper 27 bits of these small addresses are 0.
        *   **Execution Trace:**
            *   **Iteration 1 (s0=5):**
                1.  `lw s2, 4(s1)` (Address 0x4): Cache Miss (empty). Data fetched from MM, stored in Set 1. (Valid=1, Tag=0, Data=0x4_content). **Miss**
                2.  `lw s3, 12(s1)` (Address 0xC): Cache Miss (empty). Data fetched from MM, stored in Set 3. (Valid=1, Tag=0, Data=0xC_content). **Miss**
                3.  `lw s4, 8(s1)` (Address 0x8): Cache Miss (empty). Data fetched from MM, stored in Set 2. (Valid=1, Tag=0, Data=0x8_content). **Miss**
            *   **Iterations 2-5 (s0=4 down to 1):**
                1.  `lw s2, 4(s1)` (Address 0x4): Cache Hit (data found in Set 1). **Hit**
                2.  `lw s3, 12(s1)` (Address 0xC): Cache Hit (data found in Set 3). **Hit**
                3.  `lw s4, 8(s1)` (Address 0x8): Cache Hit (data found in Set 2). **Hit**
        *   **Results:**
            *   Total Misses = 3 (from Iteration 1)
            *   Total Hits = 12 (from Iterations 2-5)
            *   Total Accesses = 15
            *   `Miss Rate = 3 / 15 = 0.20` (or 20%)
        *   **Conclusion:** This example demonstrates temporal locality. Once the data is brought into the cache, subsequent accesses to the same locations result in hits, significantly improving performance.

*   **Example 5: Cache Block Conflict**
    *   **Scenario:** Same 8-word direct mapped cache (block size = 1 word), initially empty.
    *   **RISC-V Loop:**
        ```assembly
        addi s0, zero, 5  # s0 = 5 (loop counter)
        addi s1, zero, 0  # s1 = 0 (base address for loads)
        LOOP: beq s0, zero, DONE
         lw s2, 0x4(s1)   # Load word from address 0x4
         lw s4, 0x24(s1)  # Load word from address 0x24
         addi s0, s0, тИТ1  # Decrement loop counter
         j LOOP
        DONE:
        ```
    *   **Analysis:**
        *   **Total Accesses:** The loop runs 5 times. Each iteration has 2 load instructions. So, 5 * 2 = 10 total memory accesses.
        *   **Address Mapping:**
            *   Address 0x4: `[Tag (0x0...0) | Index (001) | Offset (00)]` -> maps to Set 1.
            *   Address 0x24: `[Tag (0x0...0) | Index (001) | Offset (00)]` -> maps to Set 1.
            *   **Problem:** Both addresses 0x4 and 0x24 map to the *same* cache set (Set 1).
        *   **Execution Trace:**
            *   **Iteration 1 (s0=5):**
                1.  `lw s2, 0x4(s1)` (Address 0x4): Cache Miss (empty). Data for 0x4 loaded into Set 1. **Miss**
                2.  `lw s4, 0x24(s1)` (Address 0x24): Cache Miss. Data for 0x24 loaded into Set 1, **evicting** the data for 0x4. **Miss**
            *   **Iteration 2 (s0=4):**
                1.  `lw s2, 0x4(s1)` (Address 0x4): Cache Miss. Data for 0x4 loaded into Set 1, **evicting** data for 0x24. **Miss**
                2.  `lw s4, 0x24(s1)` (Address 0x24): Cache Miss. Data for 0x24 loaded into Set 1, **evicting** data for 0x4. **Miss**
            *   **Iterations 3-5:** The pattern repeats. Each access to 0x4 and 0x24 will be a miss because the other address evicts it immediately.
        *   **Results:**
            *   Every single memory access results in a miss.
            *   Total Misses = 10
            *   Total Hits = 0
            *   `Miss Rate = 10 / 10 = 1.0` (or 100%)
        *   **Conclusion:** This illustrates a severe **conflict miss** problem in direct mapped caches. Even though the cache has enough capacity to hold both data items, they repeatedly compete for the same single cache location, leading to very poor performance. This is a primary motivation for using set-associative caches.

*   **Memory System Role:** Stores instructions, data; impacts performance.

*   **Memory Hierarchy:** Registers > Cache > Main Mem > Secondary Storage (Speed, Cost, Size tradeoff).
*   **Locality of Reference:**
    *   **Temporal:** Access recently used data again.

*   **Cache Miss:** Data not found in cache, fetch from lower level.

ЁЯУЭ **2-Mark Key Points**

1.  **What is a Memory System?** The part of a computer responsible for storing and retrieving program instructions and data.
2.  **Define Processor-Memory Gap.** The increasing speed disparity between fast CPUs and slower main memory, causing CPU stalls.
3.  **What is Temporal Locality?** The tendency for a program to access memory locations that have been recently accessed.
4.  **What is Spatial Locality?** The tendency for a program to access memory locations physically close to recently accessed locations.
5.  **Define Cache Hit.** When the requested data is found in the cache memory.
6.  **Define Cache Miss.** When the requested data is not found in the cache memory and must be fetched from a slower memory level.
7.  **What is Hit Rate?** The fraction of memory accesses that are found in the cache (Hit Rate = Number of Hits / Total Accesses).
8.  **What is Miss Rate?** The fraction of memory accesses that are not found in the cache (Miss Rate = Number of Misses / Total Accesses).
9.  **What is Cache Capacity?** The total amount of data that a cache can store, usually measured in KB or MB.
10. **What is Block Size (Cache Line Size)?** The fixed amount of data transferred between memory and cache as a single unit.
11. **What is a Valid Bit in Cache?** A bit associated with each cache entry that indicates whether the data in that entry is valid and usable (1 for valid, 0 for invalid).
12. **What is a Direct Mapped Cache?** A cache organization where each block of main memory maps to exactly one specific location (set) in the cache.
